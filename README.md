## Criação do repositório
A criação do repositório é o primeiro passo para iniciar um projeto no GitHub. Ele funciona como uma pasta online onde ficam armazenados os arquivos do projeto.

## Clone do repositório
O clone é uma cópia do repositório que fazemos para o nosso computador, permitindo trabalhar no projeto localmente.

## Criação de branch
A branch é uma ramificação do projeto principal. Ela permite desenvolver novas funcionalidades sem alterar diretamente a versão principal.

## Registro de commits
O commit é o registro das alterações feitas nos arquivos do projeto. Cada commit guarda uma mudança realizada.

## Push
O push envia as alterações do computador local para o repositório no GitHub.

## Pull
O pull serve para atualizar o repositório local com as mudanças que foram feitas no GitHub.

## Issues
Issues são usadas para registrar tarefas, problemas ou melhorias dentro do projeto.

## Milestone
Milestones são usadas para organizar e agrupar várias issues relacionadas a um objetivo.

## Project
O Project ajuda a organizar tarefas do projeto em quadros, como "a fazer", "em andamento" e "concluído".

## Pull Request
O Pull Request é uma solicitação para que as alterações feitas em uma branch sejam revisadas e adicionadas ao projeto principal.

## Merge
O Merge é o processo de juntar as alterações de uma branch com a branch principal do projeto.
Alteração feita na nova branch.
